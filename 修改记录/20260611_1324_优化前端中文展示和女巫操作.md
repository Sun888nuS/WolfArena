## 修改内容

- 前端事件流不再把未知 payload 直接显示成 `{"save":false,"poison_target_id":null}` 这类 JSON。
- 新增事件类型中文名称，例如“创建对局”“身份发放”“狼人行动”“女巫行动”“投票结算”等。
- 新增女巫行动中文描述：
  - 女巫没有使用药
  - 女巫使用了解药
  - 女巫毒杀某玩家
  - 女巫使用了解药，并毒杀某玩家
- 后端 pending action 增加 `attacked_player_id`，让前端明确显示“昨夜被袭击”的玩家。
- 女巫操作从原先的“解药 checkbox + 目标选择”改为显式单选：
  - 不使用药
  - 使用解药救人
  - 使用毒药
- 只有选择“使用毒药”时才展示毒杀目标选择框。
- 补充女巫操作样式，保持和现有行动面板一致。

## 风险与后续

- 当前前端仍在内部使用 `save`、`poison_target_id` 等 API 字段，这是接口层正常数据结构，不直接展示给用户。
- 若后续新增更多事件类型，需要继续补充中文事件名和中文描述，避免回退为技术字段。
- 后续可以把事件展示逻辑拆成独立 `eventPresenter.ts`，降低 `GamePage.tsx` 体积。
