## 修改内容

- 新增 FastAPI 后端骨架，提供 `GET /api/health` 健康检查接口。
- 新增统一配置模块，使用 `LLM_BASE_URL`、`LLM_API_KEY`、`LLM_MODEL` 支持第三方 OpenAI-compatible 中转站 API。
- 健康检查只返回 API Key 是否配置和脱敏预览，不暴露真实密钥。
- 新增 LLM provider 协议和 OpenAI-compatible provider 占位，为 Phase 2 文本 AI 接入做准备。
- 新增 React + Vite + TypeScript 前端骨架，基础页展示后端状态和中转站 API 配置状态。
- 新增 Dockerfile、`docker-compose.yml`、环境变量模板、README 和部署说明。
- README 补充 `uv` 和 `pip` 两种后端本地启动方式，并说明 Windows 下可使用 `npm.cmd`。
- 新增 Phase 0 验收文档，明确本阶段不实现游戏规则、LangGraph、LLM 实际调用和 AgentScope 语音。
- 更新 `.gitignore`，忽略本地虚拟环境、依赖目录、构建产物和缓存。

## 验证方式

- 运行 `python -m compileall app`，后端语法检查通过。
- 初次用系统 Python 导入 FastAPI 失败，原因是系统环境未安装项目依赖。
- 初次运行 `npm --version` 被 PowerShell 执行策略拦截，改用 `npm.cmd` 成功。
- 初次前端依赖安装写入用户目录 npm cache 失败，改用项目内 `.npm-cache` 成功。
- 初次 `uv sync --extra dev` 写入用户目录 uv cache 失败，改用项目内 `.uv-cache` 成功。
- 运行 `.\.venv\Scripts\python.exe -m pytest`，结果：`1 passed, 1 warning`。
- 运行 `npm.cmd run build`，前端 TypeScript 和 Vite 生产构建通过。
- 运行 `.\.venv\Scripts\python.exe -c "from app.main import create_app; app=create_app(); print(app.title)"`，输出 `WolfArena AI`。
