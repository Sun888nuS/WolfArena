# frontend/src/components

跨业务功能复用的通用组件目录。目前没有实际组件，仍作为后续抽取公共 UI 的位置。

## 适合放什么

- 多个 feature 都会使用的按钮、弹窗、状态徽章、加载态、错误提示、空状态、表单控件等。
- 不绑定具体狼人杀流程的展示组件。
- 只依赖通用 props 或 `frontend/src/types/` 中共享类型的组件。

## 不适合放什么

- 只服务游戏桌面、复盘、鉴权或语音的组件。
- 直接调用后端接口的组件。
- 内含狼人杀规则判断的组件。

## 放入规则

业务强相关组件先留在 `frontend/src/features/<feature>/`。当同一组件被两个以上 feature 稳定复用时，再提升到这里。
