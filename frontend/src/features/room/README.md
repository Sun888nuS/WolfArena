# frontend/src/features/room

房间/大厅功能预留目录。目前没有实际源码，后续可用于承载开局前的房间列表、创建房间、座位选择、邀请和游戏配置。

## 适合放什么

- 房间列表、创建房间、加入房间和座位选择页面。
- 游戏人数、角色板子、AI 难度、真人位置、模型配置等开局前设置。
- 只服务房间功能的组件、hooks、工具函数和样式。

## 需要同步的后端位置

- 开局请求字段：`backend/app/sessions/models.py`。
- 创建会话和玩家生成逻辑：`backend/app/sessions/manager.py`。
- 角色板子、人数和合法规则：`backend/app/core/rules.py`。
- 如果房间需要持久化：同步规划 `backend/app/db/` 和新的 API。

## 维护边界

房间页只负责开局前配置和房间状态展示。配置最终必须映射到后端开局请求，由后端规则核心确认是否合法。
